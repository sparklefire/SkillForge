# SkillForge 阶段二：从33条候选证据到13步有序SOP

> 系列：SkillForge 工作流拆解 · 星星之火 · NVIDIA DGX SPARK 黑客松 2026

## 这一步在干什么

上一阶段产出了33条候选证据（视频18条、手册7条、口述8条），但它们还是散的——谁先谁后、哪些是同一件事的不同说法、哪些步骤缺了来源，都还没解决。

SOP Agent 的任务是：把候选证据规划成8-15步有序操作流程，每步绑定证据，标注必要/条件属性，建立前置依赖。

## 怎么从33条变成13步

四步收敛：

1. **粗粒度拆分**：把过长的证据片段按动作边界切开
2. **细粒度合并**：把描述同一动作的多条证据合并（比如视频第190秒和手册第20页说的都是"缝标学习"）
3. **同义项归并**：不同来源用不同说法描述同一件事，归并为一个步骤
4. **依赖图检查**：确保每步的前置条件在前面已经出现，不允许"先试印再装纸"这种顺序

最终结果：13个有序步骤，其中10步必需、3步按条件执行。12步至少有两类来源印证，10步覆盖全部三类来源（视频+手册+口述）。

## 关键约束

SOP Agent 不能引用不存在的证据，也不能创建没有证据支撑的必要步骤。如果某步只有单一来源，系统会标记来源覆盖度；如果完全没有来源，该步骤会被拒绝进入Gold。

Gold SOP 由实际操作者审核确认，不是模型自己给自己打分。审核者可以修改步骤描述、调整顺序、补充条件分支。

## 多源消融发现

一个有意思的数据：只给手册，步骤覆盖80%；只给口述，覆盖90%；两样合在一起，才到100%。老师傅嘴上那些"顺便说一句"，真的是手册里写不到的。这解释了为什么单一材料永远做不出完整培训包。

## 输出

13步 Gold SOP（JSON Schema 校验通过），每步包含：步骤ID、标题、动作描述、必要工具、参数、警告、完成标准、绑定证据列表、来源类型覆盖。

详见 [Gold SOP](https://github.com/sparklefire/SkillForge/blob/main/cases/n31/gold/gold_sop.json) 和 [来源候选合成报告](https://github.com/sparklefire/SkillForge/blob/main/cases/n31/evaluations/source_candidate_synthesis_v1.json)。

这套规划方法的核心价值在于：步骤不是模型"觉得应该有"，而是证据"确实支撑"。没有来源的步骤进不了Gold，单一来源的步骤会被标记覆盖不足。
