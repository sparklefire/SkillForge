# SkillForge 阶段五：40次重算一模一样，三种演示模式不翻车

> 系列：SkillForge 工作流拆解 · 星星之火 · NVIDIA DGX SPARK 黑客松 2026

## 这一步在干什么

培训包生成完了，质检也过了，最后两个问题：结果能不能复现？现场演示会不会翻车？

SkillForge 把"可复现"当成工程指标来验证，不是嘴上说"结果稳定"，而是真跑40次看是不是每次一样。

## 40次复现测试

在 DGX Spark 的 Python 3.12 环境中：

- 直接 Gold 闭环重算：20次
- Web 现场重算：20次
- 结果：40轮全部成功，唯一 P0 语义指纹为 1 个

这意味着：同一套输入，不管跑多少次，产出的 SOP 步骤、证据绑定和质检结论完全一致。不是碰巧对了，是确定性可复现。

这个数值不包含原始视频、PDF 或录音的预处理阶段，也不代表整机 GPU 推理延迟。完整范围、耗时样本和指纹记录在 [DGX运行时基准](https://github.com/sparklefire/SkillForge/blob/main/output/evaluation/runtime_benchmark_dgx.json) 中。

## 三种演示模式

答辩现场什么情况都可能遇到：网络断了、API 挂了、设备没带。SkillForge 提供三种兜底：

1. **现场重算**：DGX 本地跑完整闭环，实时展示"发现问题→举证→修订→复检"全过程。需要网络和 Step Plan API。
2. **预处理结果**：提前跑好结果，现场展示结构化数据和修订对比。不依赖外部 API。
3. **无素材离线包**：不含原始视频和手册，只展示脱敏后的结构化成果。极端情况下也能完成演示。

每种模式有固定入口、数据边界和期望结果。不管用哪种，评委看到的"5→0、4处修订、覆盖100%"都是一样的。

## 交付清单

最终培训包包含：

- 13步 Gold SOP（三种视图）
- 13项手机检查清单
- 5题培训测验（带证据回溯）
- A4培训海报
- 80秒培训视频（15镜头、25条Evidence）
- 质检与修订审计记录
- 多源消融对比数据
- 40次可复现基准报告

## 边界说明

SkillForge 当前仍需人工审核，不是全自动决策系统。Gold 由一名实际操作者审核，第二名领域复核属于后续增强。Step Plan API 承担部分语义推断，本地大模型部署尚未作为 P0 能力。何老师参考代码只完成静态分析，以自有 N31 闭环为主。
