# SkillForge 阶段四：发现5个问题，只改4个地方

> 系列：SkillForge 工作流拆解 · 星星之火 · NVIDIA DGX SPARK 黑客松 2026

## 这一步在干什么

这是 SkillForge 最核心的环节，也是跟"让AI写一篇教程"最大的区别。

Verifier Agent 拿着规则逐条质检：步骤有没有漏？顺序对不对？工具名称有没有依据？参数是不是编的？完成标准能不能验证？每发现一个问题，它必须引用具体证据说明"为什么这里错了"——不是"我觉得不太好"，是"第7步引用了一个手册里根本不存在的工具名"。

Revision Agent 拿到问题清单后，只改出错的那几处。不是推倒重来，是精准手术。就像改作文，不是把整篇撕了重写，是只改病句。

## 真实质检过程

在操作者审核的 Gold 副本中注入5类受控错误（不在真实设备上制造危险操作）：

| 错误类型 | 具体表现 |
|---|---|
| MISSING_STEP | 缺少"介质学习"步骤 |
| MISSING_PREREQUISITE | 前置依赖断裂 |
| ORDER_ERROR | 步骤顺序颠倒 |
| UNSUPPORTED_PARAMETER | 参数无手册或视频依据 |
| UNSUPPORTED_TOOL | 工具名称无来源 |

Verifier 检测到5项严重问题，逐项引用当前步骤证据定位问题所在。Revision 执行4项局部修订：

1. 插入缺失的"介质学习"步骤（引用手册第20页和视频第190-195秒）
2. 删除无依据参数
3. 删除无依据工具
4. 恢复正确步骤顺序

复检结果：严重问题 5→0，必要步骤覆盖 90%→100%，证据覆盖 90%→100%。

## "局部"不是口号

选择性重建分析证明：这组冲突只影响7个步骤、1道题和7个视频镜头。其余步骤、题目和镜头保持不变。

这意味着：修订不会引入新错误，审核者只需要看改了哪4个地方，不需要重新审全部13步。对培训管理者来说，审核成本从"重读整份文档"降到"看4条修订记录"。

## 修订审计

每次修订都保留完整记录：修改前内容、问题描述、引用证据、修改后内容、复检结果。审核者可以逐条追溯"为什么改""依据是什么""改完对不对"。整个闭环在 Web 演示中可视化展示：点击问题编号，直接看到证据原文和修订前后对比。
