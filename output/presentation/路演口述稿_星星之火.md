## 第1段 · 真实痛点（0–20秒）｜PPT 第1–2页

**【停在封面页】**

> 设备换纸看似简单，真正困难的是正确知识散落在视频、手册和老师傅口述里。新手很容易漏掉导纸夹、介质学习和试印验收，而普通内容生成无法证明每句话从哪里来。

**操作**：展示封面，然后切到第2页（问题页），不操作网页。

**证明点**：真实主案例为汉印 N31 更换标签纸；Gold SOP 共13步，10步必需、3步条件执行。

**兜底**：若PPT不可用，直接展示 Web 页首屏并口述同一问题。

---

## 第2段 · 三类输入（20–40秒）｜PPT 第2–3页

**【第2页下半部分：三类输入】**

> SkillForge 接收专家操作视频、设备 PDF 和专家录音，保留视频时间点、PDF页码和录音时间点。原始敏感素材留在本地，公开演示只使用安全派生素材与结构化证据。

**操作**：展示第2页的三类输入区域，然后切到第3页（Agent 闭环）。

**证明点**：6段自摄安全视频、2份设备手册、12段操作者口述证据；Evidence Catalog 共154条。

**兜底**：使用离线PPT中的三类输入示意，不打开原始素材。

---

## 第3段 · DGX 本地提取（40–70秒）｜切换到 Web

**【切到浏览器 http://127.0.0.1:17860】**

> 在 DGX Spark 上，我们用原生 CUDA 对六段安全视频处理420帧，筛出50个候选时间点；语义判断仍由多源证据和Agent完成，GPU候选不会被包装成事实。整个P0不依赖Docker。

**操作**：切换到 Web 页面，指向 DGX Spark 本地视觉计算和多源证据区域。

**证明点**：NVIDIA GB10，CUDA 原生候选筛选；processed_video_count=6，sampled_frame_count=420，selected_frame_count=50。

**兜底**：若 Web 未启动，展示 PPT 第6页（DGX 指标页）。

---

## 第4段 · 发现、举证、局部修订（70–110秒）｜Web

**【Web 页面操作】**

> 现场演示使用操作者审核 Gold 构造的受控错误集，确保每次都能复现。Verifier 发现缺步骤、顺序错误、无依据工具和参数等5个严重问题；它引用证据执行4处局部修改，再质检降到0，并只失效7个步骤、1道题和7个视频镜头。

**操作**：
1. 点击"重新运行 Gold 质检与局部修订"
2. 点击一个 Evidence ID 展示安全页码/时间点
3. 指向修订前后对比、局部审计和操作者审核台

**证明点**：严重错误 5→0；4项局部修订，不整包重写；页面展示 Evidence ID 与 PDF 页码或视频/录音时间点。

**兜底**：若现场重算失败，切换 offline 模式并展示已冻结的 Gold 前后对比和审计记录。

---

## 第5段 · 培训成果（110–140秒）｜Web

**【Web 页面滚动】**

> 修订通过后，同一份可追溯 SOP 继续生成13项手机检查清单、5题测验、一页A4海报和80秒培训视频。视频15个镜头覆盖全部13步，并绑定25条去重证据。

**操作**：滚动到培训成果区，展示播放器、检查清单、测验和下载白名单。

**证明点**：13项检查清单、5题测验、一页A4海报、80秒视频；15镜头覆盖 13/13 Gold 步骤和 10/10 必要步骤。

**兜底**：若播放器受限，展示 PPT 第7页成果区和本地成片文件，不在三分钟内完整播放。

---

## 第6段 · 可复现结果（140–160秒）｜切回 PPT 第7页

**【切回 PPT 第7页】**

> 真实 Gold 评测中，步骤和证据覆盖从90%升至100%，严重错误从5降到0；DGX 上直接 Gold 与 Web 现场重算各20次，共40轮全部成功，P0语义指纹始终一致。多源消融显示，手册80%、口述90%，联合才到100%。

**操作**：切回 PPT 结果页，强调前后指标和多源消融。

**证明点**：required_step_coverage 90%→100%；evidence_supported_required_steps 90%→100%；manual_only 80%，expert_audio_only 90%，two_or_more_source_types 100%。

**兜底**：直接口述三组数值；它们同时保存在 Gold summary 与多源评测 JSON 中。

---

## 第7段 · 平台价值（160–180秒）｜PPT 第8页

**【切到 PPT 第8页（结尾页）】**

> SkillForge 不是为 N31 写死的一份教程，而是一条可迁移的 Agent 闭环：把任何设备的专家经验变成能回源、能质检、会局部自修订的培训能力。DGX Spark 让敏感视频留在现场，也让这条能力可以真正部署。

**操作**：展示结尾页并停在核心主张。

**证明点**：五个职责明确的 Agent 形成闭环；原生 Python 加 FFmpeg 即可运行完整 P0。

**兜底**：停留在 Web 首屏，复述结尾主张。
