## 1. 场景与产品定义

### SF-100 锁定唯一主案例｜P0

- [ ] 确定设备与操作任务。
- [ ] 确定目标用户和真实痛点。
- [ ] 确认素材可公开展示。
- [ ] 确认有领域人员可审核标准步骤。
- [ ] 写出20秒、100字和一页版项目介绍。

推荐条件：3–8分钟可完成、8–15个步骤、动作视觉清晰、有权威手册、有成功标准。

验收：团队任何成员能在20秒说明用户、输入、输出和价值。

### SF-101 固化演示故事｜P0

- [ ] 设计上传视频/PDF/录音的起点。
- [ ] 设计一次可控的遗漏步骤或顺序冲突。
- [ ] 设计证据定位和自动修订过程。
- [ ] 设计最终培训包展示。
- [ ] 将全流程压缩到三分钟内。

验收：演示中明确出现“发现问题→引用证据→局部修订”。

### SF-102 冻结范围｜P0

P0仅包含SOP、证据、质检、修订、检查清单、测验和可演示UI。

以下默认为P2或砍掉：数字人、多语言、全生成视频、多行业、真实机器人控制、复杂权限系统、大规模微调。

## 2. 素材与Gold数据

### SF-200 拍摄操作视频｜P0

- [ ] 准备分镜和设备初始状态。
- [ ] 横屏1080p拍摄完整任务。
- [ ] 补拍设备、工具、关键动作、完成状态和常见错误特写。
- [ ] 检查遮挡、反光、噪声和关键步骤缺失。
- [ ] 保存原始文件及授权记录。

验收：不看手册也能由视频复述完整操作。

### SF-201 准备设备手册｜P0

- [ ] 收集正确型号和版本的PDF。
- [ ] 标记相关页、警告、工具、参数、图注和表格。
- [ ] 对扫描件做OCR并核对数字、单位和型号。
- [ ] 保留原始页码映射。

验收：任一关键步骤可定位到正确页码。

### SF-202 录制专家口述｜P0

- [ ] 询问新手易错点、异常信号、完成标准和手册遗漏经验。
- [ ] 录制2–5分钟清晰单人语音。
- [ ] 校正专有名词、数字和部件名称。
- [ ] 至少提取3条手册之外的有效经验。

### SF-203 制作Gold SOP｜P0

- [ ] 人工整理8–15个步骤。
- [ ] 标注动作、对象、前置条件、工具、风险和成功标准。
- [ ] 标注PDF页码、视频时间点和专家录音时间点。
- [ ] 标注必需步骤、可选步骤和不可交换顺序。
- [ ] 由领域人员完成审核。

验收：每个必需步骤至少有一个可靠证据。

### SF-204 构造错误集｜P0

- [x] 删除一个必要步骤（已在明确标注的模拟案例中实现）。
- [x] 调换两个不可交换步骤（已在明确标注的模拟案例中实现）。
- [x] 添加一个无来源工具、参数或动作（已注入工具和参数）。
- [x] 为每种错误记录正确答案和证据（模拟基线）。

验收：质检链路能稳定发现三类错误。
