### 提交预检边界

正式提交预检必须在本地Git仓库运行：

```bash
bash scripts/check_submission.sh
```

DGX的 `~/skillforge/app` 是不携带 `.git` 的安全部署副本，只能用以下命令验证运行时部分：

```bash
bash scripts/check_submission.sh --allow-missing-git --allow-dirty
```

远端结果必须是 `DEVELOPMENT_CHECK`，其中Git工作树、跟踪敏感路径和本地 `.env` 检查标为 `SKIPPED`；不能用它替代本地正式预检。本地正式预检不允许 `--allow-dirty` 或 `--allow-missing-git`。

退出码为：`0=READY_FOR_SUBMISSION`、`1=NOT_READY`、`2=DEVELOPMENT_CHECK或READY_WITH_HUMAN_GATES`。即使返回非零，结构化报告也会先写入 `outputs/submission/submission_preflight_latest.json`，可据此逐项关闭门禁。

人工门禁通过 `bash scripts/manage_human_gates.sh status` 查看。实际完成某项工作后，用同一脚本的 `confirm` 子命令绑定当前运行单和本地证据SHA-256或安全HTTPS链接；确认不能由自动测试生成。私有记录目录权限700、文件600且由Git忽略，预检报告不会复制确认人、说明、证据定位或证据内容。证据变化、缺失、运行单哈希变化、门禁文案变化或记录权限放宽都会使确认无效并阻止正式提交。最终有声录屏必须先以固定私有文件名运行 `bash scripts/check_final_recording.sh`；该门禁只接受本地MP4，并同时校验机器QA状态、媒体哈希和当前内部策略哈希。

18项成果的角色、独立终检人角色、哈希和发布渠道由 `bash scripts/build_release_manifest.sh --verify-only` 校验，并已接入提交预检。真实姓名不进入仓库。作品页、代码仓库和最终录屏链接在上传后通过 `bash scripts/check_publication_links.sh --init` 创建私有模板，再由同一脚本匿名检查；输入和报告保存在Git忽略的 `outputs/submission/`，目录0700、文件0600，报告不会保存真实网址、域名、IP、响应正文、Authorization或Cookie。带凭证、敏感签名参数、片段、私有主机、非HTTPS跳转或最终落到非公网IP的网址直接拒绝。

团队成员和单位属于私有信息。最终名单确定后运行 `bash scripts/check_team_roster.sh --init`，仅在 `outputs/submission/team_roster.json` 填写2–5人和六类职责，目录0700、输入/QA文件0600且受Git忽略；再次运行脚本后，QA和提交预检只输出名单哈希、人数、角色数和布尔结论，不复制姓名、单位或成员编号。`READY_FOR_HUMAN_CONFIRMATION` 不会自动关闭 `TEAM_ELIGIBILITY_CONFIRMED`；确认操作会绑定当前名单SHA-256和QA状态，名单缺失、权限漂移、人数/声明/角色不完整或确认后内容变化都会使门禁失效。
