## 1. Step Coding Plan - Flash Pro

官方文档：<https://platform.stepfun.com/docs/zh/step-plan/integrations>

### 推理接口

| 用途 | 地址 |
|---|---|
| OpenAI SDK Base URL | `https://api.stepfun.com/step_plan/v1` |
| Chat Completions | `https://api.stepfun.com/step_plan/v1/chat/completions` |
| Messages | `https://api.stepfun.com/step_plan/v1/messages` |
| Anthropic SDK Base URL | `https://api.stepfun.com/step_plan` |

注意：Anthropic SDK 会自动追加 `/v1/messages`，所以其 Base URL 不能带末尾 `/v1`。

### 当前可用模型

| 类别 | 模型 | SkillForge 用途 |

| 多模态推理 | `step-3.7-flash` | 视频/图片理解、SOP规划、证据复核、复杂质检 |
| 智能路由 | `step-router-v1` | 推理服务回退或复杂请求路由 |
| 快速推理 | `step-3.5-flash-2603` | 批量抽取、改写、低成本结构化处理 |
| 快速推理 | `step-3.5-flash` | 通用Agent与代码任务 |
| 语音对话 | `stepaudio-2.5-chat` | 需要语气与人设的文本交互 |
| 语音合成 | `stepaudio-2.5-tts` | 培训视频旁白 |
| 语音识别 | `stepaudio-2.5-asr` | 专家口述和操作视频转录 |
| 实时语音 | `stepaudio-2.5-realtime` | P2实时专家访谈能力 |
| 图像生成/编辑 | `step-image-edit-2` | 海报辅助图、关键帧清理和视觉标注 |

### 密钥管理

- 真实密钥保存在项目根目录 `.env`。
- `.env` 已加入 `.gitignore`，权限应为 `600`。
- `.env.example` 只保留字段名和非敏感地址。
- 任何日志、异常、截图和演示材料均不得打印 Authorization Header。
- 当前密钥曾通过对话明文传递；若仓库要公开或人员扩大，提交前应轮换一次。

## 3. 推荐混合架构

- DGX Spark：视频转码、抽帧、OCR/Embedding、缓存、工作流执行、可选本地模型或VSS。
- Step 3.7 Flash API：多模态理解、SOP规划、证据复核和复杂质检。
- StepAudio ASR/TTS：专家录音转写和培训旁白。
- Step Image Edit：关键帧编辑、海报辅助图，不生成关键操作事实。
- 原始敏感素材默认留在DGX Spark；调用外部API前必须确认赛事与素材授权边界。
