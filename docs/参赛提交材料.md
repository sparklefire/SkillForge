# SkillForge 参赛提交材料

状态：技术内容已冻结；团队信息、官方字段和最终公开链接待参赛者按赛事规则补齐。

## 项目名称

- 中文名：匠传
- 英文名：SkillForge
- 主案例：汉印N31电子面单打印机更换标签纸、介质学习与试印验收

## 一句话定位

把专家操作视频、设备手册和口述经验，自动转化为可追溯、可验证、会自我修订的多模态培训包。

## 20秒介绍

SkillForge让Agent把操作视频、设备手册和专家口述变成有证据的SOP、清单、测验、海报和培训视频。它不只生成内容：质检Agent会发现遗漏、顺序错误和无依据参数，展示原始证据，再自动局部修订并复检。

## 约100字简介

SkillForge把视频、手册和专家口述转成有证据的培训包。Agent生成SOP，发现遗漏、顺序错误或无依据内容后，引用页码和时间点局部修订。N31实测严重问题5项降至0项，必要步骤和证据覆盖90%升至100%。

## 一页版项目说明

工业培训资料通常分散在视频、手册和熟练操作员经验中。普通内容生成工具容易遗漏关键步骤、打乱顺序，或者补写看似合理但没有依据的参数。SkillForge以统一Evidence Catalog保存PDF页码、视频时间点和录音时间点，再由五类Agent完成感知、SOP规划、内容创作、证据质检和局部修订。

主案例使用真实汉印N31设备、自摄安全视频、两份设备手册和实际操作者口述。规划层先分别保存18条视频、7条手册和8条口述候选，拆分8条过粗候选、合并8条过细候选并校验依赖，再生成13步Gold SOP、13项手机检查清单、排序/工具/风险/状态/错误判断5类证据测验、A4海报和80秒培训视频。受控错误草稿包含缺步骤、错误顺序、无依据工具和无依据参数；Verifier发现5项严重问题，Revision引用证据完成4项局部修订，复检后严重问题降为0，必要步骤和证据覆盖由90%恢复到100%。

DGX Spark承担敏感视频的本地处理、原生CUDA场景候选筛选、完整离线Agent闭环和稳定Web服务。6段安全派生视频共处理420帧并筛出50个候选时间点；51次已审核帧引用进一步合并为19个Gold步骤对齐窗口，其中12个窗口绑定41个去重DGX候选时间点。本机同时把两份手册58页拆成607个页码绑定结构块，9页中文OCR后待处理页为0。服务只监听回环地址，通过SSH隧道访问，并具有失败自动重启能力。Step 3.7语义规划和视觉复核当前使用Step Plan API，项目不会把外部API能力包装成全本地推理。

## 核心创新

1. 每个关键步骤都绑定PDF页码、视频时间点或录音时间点。
2. 把“生成”升级为“发现问题—展示证据—局部修订—再次验证”的闭环。
3. 区分设备事实、操作者经验和模型推断，模型不能覆盖人工Gold。
4. 修订只影响冲突步骤和相关成果，不进行不可审计的整包重写。
5. 敏感原始素材留在本地，公开仓库只保存脱敏结构化证据和可交付成果。
6. 置信度由来源权威性、多源佐证和负面观察共同计算；低置信度步骤自动进入人工确认，不把“未看见”当成“已完成”。

## 演示顺序

1. 展示视频、PDF和口述三类输入及Evidence定位。
2. 展示受控错误草稿中的5项严重问题。
3. 点开问题对应的手册页码、视频或录音时间点。
4. 执行4项局部修订，展示前后差异。
5. 展示选择性重建边界：7个步骤、1道题、7个镜头，其余单元不变。
6. 复检得到5→0和90%→100%。
7. 展示检查清单、测验、海报和80秒培训视频。
8. 说明DGX本地计算、隐私边界和三种演示兜底。

## 发布冻结与责任边界

`output/submission/release_manifest_v1.json` 对上述18项路演成果逐项记录相对路径、SHA-256、大小、负责人角色、独立终检角色和发布渠道。该控制清单用于冻结既有18项成果，不把自身加入哈希集合，避免循环引用。当前技术冻结摘要由以下命令重新验证：

```bash
bash scripts/build_release_manifest.sh --verify-only
```

仓库只保存六类职责和逐项角色分配，不保存团队成员真实姓名。最终成员确定后，应在Git忽略的私有提交记录中完成“姓名→角色”映射。作品页、代码仓库和最终录屏三个真实网址同样不得写入公开配置；上传后使用私有链接核验器检查。

## 已知限制

- 当前Gold只由一名实际操作者审核；第二名领域复核属于增强项。
- 稀疏关键帧不足以单独证明完整动作，视觉结果只作模型推断。
- 来源分离、粗细粒度合并、依赖排序、Gold步骤对齐跨帧窗口和本地中文PDF OCR已完成；开放式无语义规范动作发现和本地大模型推理未作为P0完成项。
- DGX公网端口不是演示依赖；正式入口使用SSH隧道，离线包和录屏兜底。
- Docker仅在后续部署本地GPU模型或NVIDIA VSS时需要。
- 最终有声录屏尚未由参赛者完成；仓库只提供内部175–180.5秒机器QA，不把该范围冒充官方视频时长要求，字幕和隐私仍需完整人工观看。
