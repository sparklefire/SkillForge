# SkillForge 最终录屏候选制作

## 1. 当前定位

该流程从公开安全的 Web 演示画面、已发布的80秒培训视频和冻结讲解词，自动生成一条接近三分钟的有声字幕候选成片。它用于减少手工录屏和剪辑风险，但只完成机器可验证部分；完整播放、字幕隐私、音画同步、节奏和官方视频规格仍须参赛者人工确认。

故事板、构建报告和数据边界分别由以下严格契约约束：

- `config/final_recording_storyboard.json`
- `schemas/final_recording_storyboard.schema.json`
- `schemas/final_recording_build.schema.json`
- `schemas/final_recording_qa.schema.json`

## 3. 安全截图规范

截图只从本地Web演示的公开展示区域采集，固定为八个PNG文件：

```text
01_overview.png
02_agents.png
03_dgx.png
04_grounding.png
05_revision.png
06_outputs.png
07_evidence.png
08_live_run.png
```

文件保存在Git忽略的 `outputs/submission/final_recording_assets/`，目录权限0700、文件权限0600。不得截入上传表单、原始素材入口、私有成员信息、绝对路径、凭证或Authorization。R05和R06在故事板中有固定裁剪区，用于只展示Evidence和修订对比的安全区域。

## 4. 自动构建

```bash
bash scripts/build_final_recording_candidate.sh
```

构建器逐场调用 `stepaudio-2.5-tts` 生成中文旁白，仅发送冻结旁白文本，不发送原始视频、PDF、录音或截图。相同模型、音色、指令和文本会命中本地私有缓存。随后使用原生Python、Pillow和FFmpeg完成字幕烧录、响度规范、九场景拼接及H.264/AAC编码，不依赖Docker。

候选成片和报告位于Git忽略的私有目录：

```text
outputs/submission/skillforge_final_recording.mp4
outputs/submission/final_recording_build.json
outputs/submission/final_recording_qa.json
```

构建报告只保存相对安全元数据和SHA-256。除既有媒体QA外，构建器会分别抽取九个场景中点，与最终成片对应时间点做差异哈希匹配，防止漏场景或顺序错乱。

## 5. 机器QA与人工门禁

可独立复验媒体机器指标：

```bash
bash scripts/check_final_recording.sh
```

机器检查覆盖：

- 175–180.5秒内部时长范围；
- 1920×1080、约30fps、H.264；
- AAC、48kHz、双声道；
- 响度、真峰值、静音和黑屏阈值；
- 九场景顺序中点匹配；
- 私有目录与文件权限。

机器通过状态只能是 `READY_FOR_HUMAN_REVIEW`。参赛者必须完整观看当前MP4，确认无隐私泄漏、字幕完整可读、旁白与画面同步、节奏可接受、文件可从头播放到结尾，并核对官方视频规则。完成这些动作后，才可绑定 `FINAL_RECORDING_REVIEW` 人工门禁；程序不得自动代替确认。
