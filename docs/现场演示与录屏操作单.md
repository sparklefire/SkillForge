# SkillForge 现场演示与录屏操作单

## 推荐顺序

1. PPT全屏打开，停在第1页。
2. 浏览器打开Web Demo，窗口宽度至少1280像素，页面缩放100%。
3. Web预先停在首屏，确认“GOLD / FINAL”和“严重错误5→0”能够出现。
4. PPT讲到40秒后切Web；110秒后展示成果；140秒切回PPT第7页。
5. 结束后不要继续操作，避免削弱结尾。

## 三种运行模式

### 现场模式｜首选

```bash
bash scripts/run_demo_mode.sh live
```

- 现场重算操作者审核Gold的结构化受控错误集。
- 不发送原始素材，不调用外部模型。
- 预期：`GOLD / FINAL`，严重错误5→0，4项局部修订。

### 预处理模式｜技术问答时使用

```bash
bash scripts/run_demo_mode.sh preprocessed
```

- 复用已经通过隐私检查的安全派生素材。
- 重跑8来源本地摄取和Gold闭环，比现场模式慢。
- 用于回答“素材解析是否真实存在”，不建议占用三分钟主路演。

### 离线模式｜最终兜底

```bash
bash scripts/run_demo_mode.sh offline
```

- 只读取仓库内冻结的Gold演示包。
- 不含原始媒体和凭证，不依赖外部API。
- 网络、DGX或现场重算异常时立即切换。

三个命令均为原生Python方案，不要求Docker。

## 开场前自动检查

```bash
bash scripts/check_pitch.sh
```

允许状态：

- `READY_WITH_HUMAN_GATES`：机器检查通过，但仍有明确的人工门禁。
- `READY_FOR_SUBMISSION`：机器检查和人工门禁都已完成。

禁止状态：`NOT_READY`。

## 录屏建议

- 分辨率：1920×1080，横屏，30fps。
- 录制范围：只录PPT、Web和必要的播放器，不录终端中的环境变量或文件路径细节。
- 系统通知：关闭。
- 浏览器：关闭密码管理、下载记录和个人书签栏。
- 麦克风：先录10秒试听，确认无爆音和明显环境噪声。
- 时长：建议最终录屏控制在2分55秒至3分00秒。
- 备份：保留一份不含现场讲解的Web静默录屏，以便后期配音。

## 失败恢复

| 现象 | 立即动作 | 继续讲法 |
|---|---|---|
| Web打不开 | 启动offline模式；仍失败则切PPT第4页 | “这里是同一份冻结Gold审计结果” |
| 重跑按钮超时 | 不重复点击，展示已有前后对比 | “现场重算有离线同构兜底，审计内容不变” |
| 视频无法播放 | 展示PPT成果页和本地文件 | “成片已通过格式、覆盖和证据边界自动检查” |
| DGX不可达 | 使用PPT第6页的已保存计算报告 | “报告记录GPU、帧数、候选数和语义边界” |
| 讲解超时 | 跳过测验细节，直接切结果页 | 保留5→0、证据、局部修订和结尾 |

## 人工门禁

- [ ] 参赛者完整观看80秒培训视频并确认旁白节奏。
- [ ] 按180秒脚本完成至少一次人工计时彩排。
- [ ] 最终录屏前复核页面和视频中没有真实面单信息、密钥或绝对路径。
- [ ] 提交前再次核对官方必交材料、外部API和团队人数规则。
