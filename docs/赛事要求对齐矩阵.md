# 第二届 NVIDIA DGX Spark 黑客松要求对齐矩阵

核对日期：2026-07-19
项目：SkillForge（匠传）  
核对结论：**选题、Agent闭环和交付形态没有方向性偏离；官方开幕PPT补齐了评分权重、开源提交和主要材料要求。参考Workshop已静态分析并按参赛者决定暂缓实跑，当前以SkillForge真实N31项目为主；团队资格、视频格式、外部API边界和现场运行规则仍未关闭。**

## 1. 本次采用的事实来源

1. [NVIDIA AI 技术专区赛事公开说明](https://nvidia.csdn.net/6a4476b3662f9a54cb87233d.html)：主题、能力方向、DGX Spark/GB10 环境、组队人数、训练营内容。
2. [NVIDIA 赛事训练营官方页](https://scrm.nvidia.cn/lp/dgx-spark-hackathon-multi-agents-20260712)：主办与合作方、主题描述、2026-07-12 训练营和四项课程入口。
3. 赛事公开说明中的日程图：作品开发与提交为 2026-07-12 至 2026-07-22，评审为 2026-07-23 至 2026-07-26，决赛路演为 2026-08-02。
4. 参赛者提供的官方开幕PPTX：第9页为六项评分权重，第10页为开源项目、600字以上说明、部署说明、技术栈、演示视频和团队合影要求；原文件只存私有目录，仓库保存SHA-256和安全结论。
5. 参赛者转交的官方完赛说明：跑通何老师参考Notebook为基础完赛标准；当前材料及复现状态见 `docs/官方参考代码复现.md`。

开幕PPT已经确认评分权重、主要提交字段和开源要求。它只要求“提交一段介绍作品的视频，清晰展示作品功能及核心亮点”，没有规定视频时长、画幅、文件格式或托管平台；也没有明确外部API许可和决赛现场运行方式。因此状态仍为 `OFFICIAL_DETAIL_REQUIRED`，但待核项已由6项缩减为3项。

### 2026-07-19 公开可访问性复核

- 匿名交互访问训练营页时可以看到“黑客松赛事规则说明”课程标题，但该课程行没有公开讲义下载控件。
- 页面可见的3个“讲义下载”控件分别与后续3项技术课程对齐，不是赛事规则讲义；另有1个代码下载控件同样属于技术课程。
- 再次复审回放页和上述4个资料入口后，资料类型结论未变化；公开回放配置没有返回匿名可用的播放地址，不能据此取得或推断规则细则。
- 页面没有公开赛事规则附件链接，也没有观察到单独返回规则正文的公开接口；没有进行报名、登录或用户校验。
- 获取规则原文仍需参赛者完成官方报名验证，或从赛事群、报名邮件、主办方取得材料。这里的结论是“公开匿名访问没有取得细则”，不是“赛事没有这些要求”。
- 结构化核验快照保存在 `config/official_rules_status.json`，由严格Schema和提交预检校验：当前公开确认8项、参赛者官方材料确认4项、待官方细则3项，不能借机器检查自动关闭 `OFFICIAL_RULES_VERIFIED` 人工门禁。
- 私有六项审核表已经绑定官方PPTX并填入逐项草稿；原文件、规则正文和人工确认不进入Git。远端开源发布策略和3项未知规则未解决前，状态保持 `PENDING_INPUT`。

1. 完整项目上传到GitHub、码云等开源平台，并以URL提交。
2. 600字以上项目说明，包含作品特点、核心亮点、技术实现、架构和优化方案。
3. 部署说明，明确本地算力如何部署智能体和优化大模型。
4. 技术栈说明，列明NVIDIA SDK、NVIDIA及Stepfun相关模型。
5. 作品演示视频，清晰展示功能和核心亮点。
6. 团队合影。
7. 跑通何老师参考代码并保存实际复现证据，作为基础完赛标准。

## 4. 是否存在偏离

### 没有偏离的部分

- 不是单纯“生成一段视频”，而是完整的多模态 Agent 自动化工作流。
- “发现问题 → 展示证据 → 自动局部修订 → 展示前后对比”已经真实实现，且有 Gold 指标支撑。
- 真实 N31 案例具有设备、手册、视频、口述、成功标准和受控冲突，难度足以证明通用流程。
- DGX Spark 已经实际参与安全视频的本地 GPU 计算，原始敏感素材没有被伪装成云端公开数据。
- Docker 不是公开赛事要求。当前原生 Python + FFmpeg + CUDA 路线不会因为不使用 Docker 而偏题。

### 尚未确认的硬风险

1. **团队人数**：公开规则要求 2–5 人。当前不知道报名队伍是否已经满足该条件。
2. **参考代码边界**：当前只有Notebook和PPT，缺少培训环境预装的Ollama、ComfyUI、OpenClaw、模型与控制脚本；静态分析已完成，参赛者决定暂不复现，资源集中到真实N31项目。
3. **详细提交规则**：视频时长与格式、外部API边界、现场运行方式仍没有官方明文。
4. **开源发布**：官方要求完整项目公开URL，但当前没有远端；公开前必须完成密钥、原始素材、第三方材料和私有状态边界复核。

### 不构成违规、但可能影响得分的风险

1. **DGX 叙事仍可加强**：420 帧原生 CUDA 计算证明了真实使用，但语义规划和视觉复核主要来自外部 Step Plan。若评审偏重“本地模型能力”，当前方案会显得 DGX 更像安全预处理节点。
2. **最终人工门禁未关闭**：80秒培训成片完整观看、180秒真人彩排、有声录屏、团队资格和官方规则五项仍为 `PENDING`；提交预检不会替人确认。
3. **现场公网入口未打通**：DGX 服务在 `0.0.0.0:7860` 启动及进程重启后回环访问均成功；公网普通请求返回 502，绕过本机代理后连接建立但收到空响应，而且请求没有到达 Uvicorn 日志。问题位于公网入口或中间网络层，离线和 SSH 端口转发兜底不受影响。

## 5. 当前建议

### 截止提交前必须完成

1. 参赛者确认报名队伍为 2–5 人，并把最终成员姓名和分工写入提交材料。
2. 不再主动获取或复现何老师Workshop；若后续官方明确要求实际运行证据，再取得合法bundle并在独立DGX用户目录安全复现，禁止原样执行清端口和不安全LAN认证单元。
3. 向组织方补问视频时长/格式、外部API和现场运行三项未知规则。
4. 完整观看80秒成片并记录通过/需修改。
5. 按冻结脚本完成一次180秒真人彩排，再录制有声兜底视频。
6. 明确授权后创建公开代码仓库并推送安全交付范围，同时撰写赛事征文和准备团队合影。
7. 提交前逐项验证链接、无密钥、无私有原始素材、作品名称和成员信息一致。

仓库内 `bash scripts/check_submission.sh` 已自动覆盖无密钥、无跟踪私有素材、成果格式/哈希、说明文档、命令、项目身份和官方规则核验快照；公开链接、成员和官方规则原文仍需人工确认。

### 按已确认评分应优先追加

- 先完成何老师参考代码的本地LLM + ComfyUI + Agent Skill基线，它同时强化25%的技术深度和15%的平台适配性。
- 若时间允许，再让本地VLM/LLM对SkillForge候选窗口做二次复核，并把结果限制为 `MODEL_INFERENCE`，不覆盖手册和人工Gold。
- 参考Notebook本身采用原生运行，不需要为了复现而先处理Docker；只有选定必须容器化的本地模型或VSS时再处理Docker权限。

## 6. 最终判断

**整体方向高度对齐，核心闭环甚至比普通内容生成项目更贴合“持续优化”。当前最需要防止的不是选题跑偏，而是资格和提交细节未确认，以及 DGX 的语义价值在三分钟路演中不够重。**

路演应保持以下表述：

> SkillForge 在 DGX Spark 上本地解析敏感操作素材并执行可审计的 GPU 候选筛选；多源 Agent 把视频、手册和专家经验变成可追溯培训包，再通过证据质检和局部修订把严重问题从 5 项降到 0 项。

不要表述为“全部模型均在 DGX 本地运行”，因为当前事实并非如此。
